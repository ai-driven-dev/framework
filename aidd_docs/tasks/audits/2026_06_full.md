# Codebase Audit: AIDD CLI (full)

Structurally sound hexagonal CLI; no systemic rot. Real work concentrates in one perf hot-path bug, a dependency-policy + CVE cleanup, security hardening around untrusted catalogs, and coverage gaps on a few critical use-cases.

- **Date**: 2026-06-11
- **Scope**: `/aidd/cli` — full codebase, 6 pillars (UI N/A)
- **Health**: fair — critical findings exist but all isolated, trivial-to-medium fixes; architecture + clean-code are healthy.
- **Findings**: 7 critical, 29 warning, 16 minor

Health: `good` = no critical; `fair` = critical but isolated/addressable; `poor` = systemic.
Severity follows the shared rubric (🔴 broken/exploitable now · 🟡 real debt/risk · 🟢 nit). Note: the `tests` pillar rates 🔴 as "critical-path untested" (risk, not proven-broken) — ranked accordingly in Top actions.

## Top actions (ranked by impact)

1. **Kill the hot-path network block** (🔴 perf, `cli.ts:65`) — the single highest user-facing win (~600ms off every first command). Then re-capture `perf-baseline.json` (🟡) so the gate can catch regressions. → `aidd-dev:07-refactor` (perf axis).
2. **Dependency hygiene** — one pnpm `override` fixes both prod fast-uri CVEs (🔴, effort S); decide the runtime-dep policy (amend to 6, or shrink) (🔴); bump dev toolchain to clear the remaining CVEs (🟡). → `aidd-dev:07-refactor` (deps) + a policy decision.
3. **Security hardening of the untrusted-catalog path** (🟡×4) — npm-spec validation (pnpm injection), per-source trust gate, prototype-pollution guard in `deepMerge`, lstat-skip symlinks. Small fixes, real exploit surface. → `aidd-dev:07-refactor` (security axis).
4. **Close the critical-path test gaps** (🔴 tests) — `sync-status`, `marketplace-cache`, the 4 `install-*` use-cases; un-exclude `auth/**` from coverage; this also lifts coverage past the 85/90 gate. → `aidd-dev:06-test`.
5. **Clean-code sweep** (🟡) — 3 silent catches → warn; delete the knip-confirmed dead code (7 errors + 4 fns + 1 alias + 1 type + 1 devDep); de-dup the 9× `isAiToolId` idiom + the JSONC stripper. Mechanical, low-risk. → `aidd-dev:07-refactor` (cleanup axis).
6. **Thin-command leaks** (🟡 arch) — extract `UpdateAiToolsUseCase` / `UpdateIdeToolsUseCase` so `ai/ide update` stop orchestrating in the command layer. → `aidd-dev:07-refactor` (architecture axis).
