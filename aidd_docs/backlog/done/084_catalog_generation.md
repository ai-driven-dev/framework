# 084: Generate aidd_docs/CATALOG.md from manifest at install/uninstall/init time

## Context

`aidd_docs/CATALOG.md` is installed as a docs file from the framework. It contains markdown links pointing to framework-repo-relative paths (e.g. `[alexia.md](../agents/alexia.md)`) that do not exist in the installed project. After installation, agents go to `.claude/agents/`, `.github/agents/`, etc. depending on installed tools. This causes `aidd doctor` to report dozens of false warnings from CATALOG.md.

The fix: the CLI generates `CATALOG.md` from the manifest after every `init`, `install`, and `uninstall`. The framework's `CATALOG.md` is excluded from docs installation — the CLI owns the output entirely.

## Scope

- Add pure domain function `generateCatalogContent(manifest, docsDir): string`
- Add thin app-layer helper `writeCatalog(manifest, docsDir, projectRoot, fs): Promise<void>`
- Exclude `CATALOG.md` from docs installation in `InitUseCase`
- Call `writeCatalog` after `manifestRepo.save()` in `init`, `install`, `uninstall`
- CATALOG is NOT tracked in manifest (generated, not installed)

```markdown
## Docs

| File |
|------|

```
