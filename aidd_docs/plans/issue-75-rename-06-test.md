## Objective

Rename the `06-test` skill in the `aidd-dev` plugin to `06-write-tests` so its name accurately reflects what it does (creates/writes tests) instead of suggesting it runs tests. Update every cross-reference in the repo to the new slug and regenerate the auto-generated catalog so it stays consistent.

The skill behavior is unchanged. This is a pure rename / reference-update task.

### New naming

| Field                  | Old                | New                       |
| ---------------------- | ------------------ | ------------------------- |
| Directory name         | `06-test`          | `06-write-tests`          |
| Skill `name` frontmatter | `aidd-dev:06:test` | `aidd-dev:06:write-tests` |
| Slug used in prose     | `06:test`, `06-test` | `06:write-tests`, `06-write-tests` |

The numeric position `06` is preserved to avoid renumbering the other skills.

- [ ] The directory `plugins/aidd-dev/skills/06-test/` no longer exists; `plugins/aidd-dev/skills/06-write-tests/` exists with the same contents (`SKILL.md`, `actions/`, `evals/`).

## Risks and Discrepancies Found During Planning

1. **`scripts/summarize-markdown.mjs` does not exist in this repo.** The CATALOG header claims it is auto-generated by that script, but a `find` for `summarize-markdown*` returns zero matches, and the only `.mjs` file under `plugins/` is `plugins/aidd-dev/scripts/list-rules.mjs`, which is unrelated. The issue instructs us to "regenerate CATALOG.md by running the summarize script" — that command will fail. Decision (recorded): the implementer should update `plugins/aidd-dev/CATALOG.md` by hand (it is a small mechanical rewrite — five `06-test` tokens) and surface the missing-script discovery as a follow-up. Do NOT introduce a new script in this PR; that is scope creep.
2. **User-supplied file path is wrong.** The issue lists `plugins/aidd-dev/skills/aidd-context/skills/02-project-init/assets/README.md`. That nested path does not exist. The actual file is `plugins/aidd-context/skills/02-project-init/assets/README.md` (sibling plugin). Line numbers 42, 100, 116 are correct for that real path and contain the references to update.
3. **No tests exercise this skill name.** A repo-wide search did not find a CI step that asserts the skill's identifier, so the only validation is the grep check.
