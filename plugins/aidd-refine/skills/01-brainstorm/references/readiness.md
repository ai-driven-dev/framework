# Readiness

Use readiness internally to decide whether to continue or finalize.

- Track each relevant core point and transversal branch as settled, open, or deferred.
- Add a reason for each deferred item.
- Continue probing while an open item can still change the artifact.
- Finalize when unresolved items are explicit assumptions and the user accepts them.
- Never show status, coverage, matrix, or table unless the user asks for an audit view.
